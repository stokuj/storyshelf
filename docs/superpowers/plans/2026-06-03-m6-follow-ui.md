> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the UI for the existing `UserFollow` backend — a Follow/Unfollow button and follower/following counts on the public profile, plus dedicated followers/following list subpages.

**Architecture:** Small backend additions expose follow state on existing endpoints (counts + `is_following` on the profile serializer; an enriched list serializer returning the other side of each relation). **Tech Stack:** Django 6 + DRF (backend), SvelteKit 2 + Svelte 5 runes + Tailwind v4 (frontend), Playwright (E2E), drf-spectacular (OpenAPI contract).

### Model gotcha — `UserFollow` related_names are REVERSED vs intuition
`backend-django/users/models.py`:
```python
class UserFollow(models.Model):
    follower  = models.ForeignKey(User, on_delete=CASCADE, related_name="following_set")
    following = models.ForeignKey(User, on_delete=CASCADE, related_name="follower_set")
    followed_at = models.DateTimeField(auto_now_add=True)
```
So for a `User` instance `u`:
- `u.follower_set` → rows where `u` is the **following** target → **people who follow u** → `followers_count`.
- `u.following_set` → rows where `u` is the **follower** → **people u follows** → `following_count`.

### Count annotation gotcha
Annotating two `Count()` over different relations in one queryset multiplies rows (JOIN fan-out) and yields wrong numbers. **Always pass `distinct=True`**: `Count("follower_set", distinct=True)`.

### What must NOT break
- `POST /api/users/{handle}/follow/` keeps returning `FollowSerializer` shape (`follower_handle`, `following_handle`). `UserFollowTest` depends on it. **Only the list view changes serializer.**
- Any API change requires regenerating the OpenAPI snapshot (`make regenerate-openapi`) or `config.tests.test_openapi_schema` fails. This is its own task at the end of the backend work.

class UserProfileView(generics.RetrieveAPIView):
    """
    Publiczny profil użytkownika po handle. Zwraca 404 jeśli profil jest prywatny
    i requestujący nie jest właścicielem.
    """

    permission_classes = (permissions.AllowAny,)
    serializer_class = UserProfileSerializer
    lookup_field = "handle"

    def get_queryset(self):
        # related_name is reversed: follower_set = people who follow this user,
        # following_set = people this user follows. distinct=True avoids JOIN fan-out.
        return User.objects.annotate(
            followers_count=Count("follower_set", distinct=True),
            following_count=Count("following_set", distinct=True),
        )

    def get_object(self):
        user = super().get_object()
        if not user.profile_public and user != self.request.user:
            raise Http404("No User matches the given query.")
        return user
```

class FollowUserSerializer(serializers.Serializer):
    """
    Serializes one side of a UserFollow row as a compact user card.
    Which side depends on context["follower_view"]:
      True  -> show the follower (used by the "followers" list)
      False -> show the following (used by the "following" list)
    """

    handle = serializers.SerializerMethodField()
    display_name = serializers.SerializerMethodField()
    avatar_url = serializers.SerializerMethodField()
    followed_at = serializers.DateTimeField(read_only=True)

    def _target(self, obj):
        return obj.follower if self.context.get("follower_view") else obj.following

    def get_handle(self, obj):
        return self._target(obj).handle

    def get_display_name(self, obj):
        return self._target(obj).display_name

    def get_avatar_url(self, obj):
        user = self._target(obj)
        if user.avatar:
            request = self.context.get("request")
            return request.build_absolute_uri(user.avatar.url) if request else user.avatar.url
        return None
```

## Task 5: FollowButton component (optimistic)

**Files:**
- Create: `svelte-frontend/src/lib/components/FollowButton.svelte`

- [ ] **Step 1: Create the component**

Create `svelte-frontend/src/lib/components/FollowButton.svelte`. It owns optimistic state for the follow flag and the follower count, reverts on real failure, and shows a toast. It resyncs when the `handle` prop changes (SvelteKit reuses the component across navigations — see the prop-seeded-state gotcha, commit 4c968a7).

```svelte
<script lang="ts">
	import { Button } from '$lib/components/ui/button';
	import { toast } from 'svelte-sonner';
	import { followUser, unfollowUser } from '$lib/api/follow';

	interface Props {
		handle: string;
		isFollowing: boolean;
		followersCount: number;
		/** Bubbles the optimistic follower count up to the profile header. */
		onCountChange?: (count: number) => void;
	}
	let { handle, isFollowing, followersCount, onCountChange }: Props = $props();

	let following = $state(isFollowing);
	let count = $state(followersCount);
	let pending = $state(false);

	// Resync when navigating to a different profile (props change, component reused).
	$effect(() => {
		following = isFollowing;
		count = followersCount;
	});

	async function toggle() {
		if (pending) return;
		pending = true;
		const wasFollowing = following;
		// optimistic
		following = !wasFollowing;
		count += wasFollowing ? -1 : 1;
		onCountChange?.(count);

		const { error } = wasFollowing
			? await unfollowUser(fetch, handle)
			: await followUser(fetch, handle);

		// 409 (already following) / 404 (not following) == desired end state, keep optimistic.
		const realError = error && error.status !== 409 && error.status !== 404;
		if (realError) {
			following = wasFollowing;
			count += wasFollowing ? 1 : -1;
			onCountChange?.(count);
			toast.error('Something went wrong. Please try again.');
		}
		pending = false;
	}
</script>

<Button
	variant={following ? 'outline' : 'default'}
	size="sm"
	disabled={pending}
	onclick={toggle}
>
	{following ? 'Unfollow' : 'Follow'}
</Button>
```

		// Followers list now empty of authUser.
		await page.goto(`/u/${guestUser.handle}/followers`);
		await expect(page.getByText(`@${authUser.handle}`)).toHaveCount(0);
	});
});
