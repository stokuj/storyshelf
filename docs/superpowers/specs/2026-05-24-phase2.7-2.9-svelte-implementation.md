# Spec: Phase 2.7–2.9 — Svelte Frontend Implementation

> Data: 2026-05-24
> Cel: Działający frontend SvelteKit od zera do kompletnej aplikacji.
> Źródło: `svelte(wideframe)/handoff/` — CLAUDE.md, prompts 01–11, docs 01–05.

---

## Kontekst

`svelte-frontend/` ma już:
- SvelteKit 2 scaffold z TypeScript, Tailwind v4, ESLint, Prettier, Vitest
- `_client.ts` — kompletny API client z obsługą 401 + silent refresh
- `user.ts` — API wrapper dla user endpoints
- `types/api.generated.ts` — typy wygenerowane z OpenAPI
- `utils.ts` — `cn()`, `config.ts` — env constants
- `hooks.server.ts` — cookie passthrough
- Pusty `+layout.svelte`, `+page.svelte`, `+error.svelte`

Brakuje: shadcn-svelte, theme, AppShell, atomów, API wrapperów (books/characters/ai/shelves), schematów Zod, tras, mock fixtures.

Stack (non-negotiable):
- Svelte 5 runes only — `$state`, `$derived`, `$props`, `$effect`. Żadnego `export let`, `$:`.
- Tailwind v4 CSS-first (`@theme` w `app.css`, bez `tailwind.config.js`)
- shadcn-svelte dla prymitywów
- superforms + zod dla formularzy
- @xyflow/svelte dla grafu relacji
- mode-watcher dark mode

## Phase 2.7 — Foundation (prompts 01–04)

### 2.7.1 Setup & design system

**shadcn-svelte install:**
```
pnpm dlx shadcn-svelte@latest init
pnpm dlx shadcn-svelte@latest add button card dialog tabs avatar input label switch radio-group dropdown-menu sheet sonner tooltip badge separator skeleton command popover
```

**`src/app.css`** — `@theme` block z tokenami z `handoff/01-design-system.md`:
- Kolory: `--color-paper`, `--color-ink`, `--color-accent`, `--color-accent-soft`, `--color-accent-ink`, `--color-rule`, `--color-rel-*`
- Font display: Newsreader (serif), body: Public Sans (sans)
- Wariant dark: `.dark` override
- `@layer base`: body background, text, font-family, selection color

**`src/app.html`** — `<link>` dla Newsreader + Public Sans z Google Fonts.

**`src/routes/+layout.svelte`** — `ModeWatcher` + `Toaster` + `AppShell`.

### 2.7.2 AppShell i atomy

**`src/lib/components/shell/`:**
- `AppShell.svelte` — sticky top nav: brand, search trigger (`Cmd+K`), nav links (Discover / My shelf), prawy UserMenu + `<slot />`
- `UserMenu.svelte` — avatar trigger, DropdownMenu: Profile / Settings / Sign out. Dark mode toggle via mode-watcher.

**`src/lib/components/book/`:**
- `BookCover.svelte` — aspect-2/3, fallback striped placeholder gdy `cover_url` null. Props: `book`, `size: 'xs'|'sm'|'md'|'lg'|'xl'`
- `BookCard.svelte` — cover + title + author + genre badge
- `BookGrid.svelte` — responsive grid 5/3/2 cols, props: `books`
- `BookGridSkeleton.svelte` — 10 skeletonów

**`src/lib/components/character/`:**
- `CharacterAvatar.svelte` — kołowy, fallback monogram inicjałów. Props: `character`, `size`
- `RelationBadge.svelte` — kolorowy dot + label wg `kind`

**`src/lib/components/ai/`:**
- `AIBadge.svelte` — `Sparkles` icon + "AI" pill, `bg-accent-soft text-accent-ink`
- `AIPanel.svelte` — pusty wrapper, dashed accent border + title slot

### 2.7.3 Typy, schematy, API wrappery

**`src/lib/types/`** — pliki wg `handoff/02-data-model.md`:
- `book.ts` — `Book`, `Author`, `Genre`, `Tag`, `Review`, `ShelfEntry`, `PaginatedList<T>`
- `character.ts` — `BookCharacter`, `BookRelation`, `RelationType`
- `user.ts` — `User`, `UserSettings`
- `ai.ts` — `AIExtraction`, `ExtractionStatus`, `ConfidenceSummary`
- Re-export z `src/lib/types.ts`

**`src/lib/schemas/`** — Zod schematy (typy derivowane z schematów):
- `book.ts` — review schema (rating 1–5, body 10–5000 chars)
- `user.ts` — change email, change password, update profile
- `settings.ts` — AI preferences, visibility

**API wrappery** (wg `handoff/03-api-contract.md`):
- `src/lib/api/books.ts` — `listBooks`, `getBook`, `getReviews`, `createReview`, `searchByCharacter`
- `src/lib/api/characters.ts` — `getBookCharacters`, `getBookRelations`, `getCharacter`, `getCharacterAppearances`, `updateCharacter`
- `src/lib/api/ai.ts` — `runExtraction`, `getExtraction`, `refreshExtraction`, `verifyCharacter`, `verifyRelation`, `rejectCharacter`
- `src/lib/api/shelves.ts` — `getMyShelves`, `addToShelf`, `removeFromShelf`

**Mock backend:**
- `PUBLIC_API_URL=MOCK` → używaj fixture
- Fixtures w `src/lib/api/__fixtures__/`: 2 książki, 8 postaci dla jednej, 6 relacji, 1 user, 1 settings, 1 extraction

### 2.7.4 Trasa `/discover`

**`src/routes/discover/+page.server.ts`** — `load()` parsuje search params (`q`, `genre`, `sort`, `page`), wywołuje `listBooks(fetch, params)`.

**`src/routes/discover/+page.svelte`** — grid + filterbar. URL jako source of truth, filtry piszą przez `goto`.

**`src/lib/components/discover/`:**
- `FilterBar.svelte` — genre dropdown, sort dropdown, character combobox
- `ContainsCharacterCombobox.svelte` — Command-based, debounced 250ms, ustawia `?character=`

**Behavior:**
- "Load more" paginacja
- Empty state gdy brak wyników
- `<svelte:head>` z tytułem "Discover — Storyshelf"

**Acceptance 2.7:**
- `pnpm check` 0 errors
- `pnpm lint` 0 warnings
- `/discover` renderuje fixture books w MOCK mode
- Filtry → URL update → lista odświeża
- Dark mode toggle działa, brak flash
- Responsive: 320px / 768px / 1280px

## Phase 2.8 — Books, Characters, AI (prompts 05–07)

### 2.8.1 Book detail `/books/[slug]`

**`src/routes/books/[slug]/+page.server.ts`** — `getBook(slug)` + `getReviews(slug, page=1)`.

**`src/routes/books/[slug]/+page.svelte`** — 3-kolumnowy layout: cover+CTA | opis+recenzje | AI panel.

**`src/lib/components/book/`:**
- `BookHeader.svelte` — title, author, year, genres
- `BookActions.svelte` — split button "Want to read" z DropdownMenu. Optimistic UI: `addToShelf` + rollback on error
- `BookDescription.svelte` — typography-tuned, `font-display italic` dla pulled quotes
- `ReviewList.svelte`, `ReviewItem.svelte` — lista recenzji z paginacją
- `ReviewComposer.svelte` — superforms + Zod schema (rating 1–5, body 10–5000 chars)
- `RatingStars.svelte` — interaktywny (+ read-only mode)

**Loading:** per-sekcja skeleton, recenzje nie blokują całej trasy.

### 2.8.2 AI Cast Panel

**`src/lib/components/ai/AICastPanel.svelte`** — state machine przez `$state`:
- `idle` — explainer + przycisk "Generate ✨"
- `pending` — 8 skeleton rows + "Reading the book…"
- `ready` — lista `CharacterRow`, "See all (n)" → `/books/[slug]/characters`, quick-action chips
- `failed` — error block + Retry

**`src/lib/components/ai/CharacterRow.svelte`** — avatar + name + role + AIBadge. Menu: Verify → `verifyCharacter(id)`, Reject → Dialog z 4-opcjowym formularzem.

**`src/lib/components/ai/RelationGraphDialog.svelte`** — full-screen Dialog z `@xyflow/svelte SvelteFlow`:
- Node = CharacterAvatar + label
- Edge color wg `kind` (`--color-rel-*`), dashed dla "enemy", thick dla "romance"
- Klik na node → navigate do character page
- Filter chips: all / family / romance / ally / enemy
- Keyboard fallback: `<ul>` relacji obok grafu dla screen readerów
- Lazy load `@xyflow/svelte` (code-split)

### 2.8.3 Character pages

**`/books/[slug]/characters`** — 2-kolumnowy layout:
- Lewa: filtrowalna lista (search by name, filter by role)
- Prawa: detail wybranej postaci (`?focus=`)
- Mobile: stacked

**`/books/[slug]/characters/[charId]`** — in-book character page:
- Header: avatar xl, name (display font), AKA, role, tag badges
- 2-kolumnowy: bio+quotes | relations+appearances
- Edit button → Sheet z superforms (tylko dla uprawnionego usera)

**Acceptance 2.8:**
- `/books/dune` renderuje metadata, opis, recenzje
- Review post → toast "Review posted"
- AI Generate → `pending → ready` w MOCK mode
- Graf relacji: 6 fixture relacji, klawiszowy fallback
- Verify/Reject aktualizuje local state optimistycznie
- 3 cols ≥1024px, stack poniżej

## Phase 2.9 — Profile, Settings, Polish (prompts 08, 09, 11)

### 2.9.1 Profil `/u/[handle]`

**`src/routes/u/[handle]/+page.server.ts`** — `getUser(handle)` + `getUserShelves(handle)`.

**Komponenty:**
- Header: avatar, display name, handle, joined date, follower count, top genres
- Reading challenge: progress bar `X / Y` wg `user.reading_goal`
- Recently read: rząd 6 coverów ze star ratings
- AI sidecar (dashed accent border, prawa kolumna): recent extractions, inline settings preview
- Visibility: `private` → "This profile is private." dla nie-właściciela

### 2.9.2 Settings `/settings/*`

**Layout** `src/routes/settings/+layout.svelte` — left rail:
- Account (default) / Profile & privacy / Notifications / AI preferences / Reading & shelves / Data & export / Danger zone

**`/settings` — Account:**
- Avatar: drag-and-drop `<input type="file">` + crop dialog + `uploadAvatar`
- Display name / username / email — superforms, auto-save on blur
- Email change → verify-email modal
- Password (current/new/confirm) ze strength bar

**`/settings/profile` — Profile & privacy:**
- 3 RadioGroup cards: Public / Friends only / Private
- Switch toggles: real-name visibility, activity feed, followed-characters, AI-learning, SEO indexing

**`/settings/notifications`** — email + push toggles per event type.

**`/settings/ai`** — tone radio, default spoiler limit radio, cite quotes radio, chat history retention slider (7/30/90/forever).

**`/settings/data`** — Export data (link do download) + Pause account + **Delete account** (wymagany handle confirmation w Dialog, `DELETE /users/me` → signout → redirect `/`).

### 2.9.3 Polish pass

- Empty states dla każdej listy: friendly copy + ikona + CTA
- `+error.svelte` root + per route-group (books, u/[handle]): retry + go-home
- 404: literacka linia w root `+error.svelte`
- Keyboard nav: wszystkie trasy bez myszy, focus trap w dialogs, skip-to-main-content
- A11y: alt na obrazach, `aria-hidden` na dekoracyjnych ikonach, WCAG AA contrast
- Performance: `loading="lazy"` na coverach poniżej fold, code-split `@xyflow/svelte`, `prerender=true` na login/signup
- Meta: `<svelte:head>` per route, OG tags na book i profile, `manifest.webmanifest`
- Auth: `/login` i `/signup` — pełne formularze superforms + Zod, `POST /auth/login/` i `/auth/register/`

**Acceptance 2.9:**
- Lighthouse mobile: Performance ≥ 85, A11y ≥ 95, Best Practices ≥ 95
- `pnpm check` 0 errors, `pnpm lint` 0 warnings
- Delete account wymaga wpisania handle, nigdy one-click
- Keyboard walkthrough: Discover → Book → AI panel → Verify → Profile → Settings
