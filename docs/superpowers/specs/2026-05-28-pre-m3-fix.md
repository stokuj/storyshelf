# Spec: PRE-M3-FIX — Project Cleanup Before Milestone 3

**Date:** 2026-05-28  
**Status:** Approved  
**Goal:** Bring the codebase to a clean M2-complete state. Remove all code that doesn't belong in M1 or M2. No code written for future milestones. No AI/LLM infrastructure. No Celery/RabbitMQ/Redis.

**Source of truth:** `docs/GOAL.md`

---

## Context

During M2 implementation, extra code accumulated that belongs to future milestones (M3–M5) or was explicitly cut from MVP:

- `analysis/` Django app — AI/NLP character extraction (cut from MVP per GOAL.md)
- `reviews/` Django app — M4 feature
- `shelf/` Django app — M3+M5 features
- Celery workers, RabbitMQ, Redis, Flower — explicitly excluded per GOAL.md
- Frontend AI components, character pages, review components — cut from MVP
- `/settings/profile` privacy toggle — M1 feature, broken (no handler)

---

## Scope

### 1. Backend — Remove Django apps

**Remove entirely** (models, migrations, views, serializers, signals, tests, admin, apps.py, urls.py):

| App | Reason |
|-----|--------|
| `analysis/` | AI/LLM — cut from MVP |
| `reviews/` | M4 — not yet |
| `shelf/` | M3 (ShelfEntry) + M5 (Shelf, ShelfMembership) — not yet |

Steps per app:
- Delete directory `backend-django/<app>/`
- Remove from `INSTALLED_APPS` in `config/settings/base.py`
- Remove `include("<app>.urls")` from `config/urls.py`

### 1b. Backend — Clean remaining apps of removed-app coupling

Removing the apps above breaks imports and code paths in `users/`, `books/` and `config/`. These must be removed too (verified by grep — not just `urls.py`):

**`config/urls.py`:**
- Remove `from reviews.views import BookReviewListCreateView`
- Remove routes: `/api/books/<id>/reviews/`, `/api/shelf/`, `/api/reviews/`, `/api/` → `analysis.urls`

**`config/` Celery code** (spec §4 only covers Docker; the Django side lives here):
- Delete `config/celery.py`
- Revert `config/__init__.py` to empty (remove `from .celery import app`)
- Remove `CELERY_*` settings block from `config/settings/base.py` (broker, result backend, task routes, retries)

**`users/` app:**
- `users/views.py`: remove `PublicUserShelvesView` (Shelf — M5) and `PublicUserRecentlyReadView` (ShelfEntry — M3); remove `from shelf.models import Shelf, ShelfEntry` and `from shelf.serializers import ShelfSerializer`
- `users/urls/users.py`: remove routes `<handle>/shelves/` and `<handle>/recently-read/` and their view imports
- `users/exporters.py` (`build_user_export_zip`): remove the `shelf_entries.json` and `reviews.json` sections; keep `user.json` + `follows.json`. **`DataExportView` stays** (user-domain account export, M1) — verify it still works after stripping
- `users/tests/test_data_export.py`: drop shelf/review assertions, keep account+follows export coverage
- `users/tests/test_public_profile.py`: remove `from shelf.models import Shelf, ShelfEntry` and shelf-related assertions

**`books/` app:**
- `books/views.py`: remove the `contains-character` action (`@action url_path="contains-character"`) and `from analysis.models import BookCharacter` (GOAL.md M2 filters do not include contains-character)
- `books/admin.py`: remove `from analysis.admin import analyze_selected_books` and the admin action registration
- `books/tests/test_contains_character.py`: delete entirely
